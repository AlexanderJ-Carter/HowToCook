# 如何贡献

感谢你对本仓库的兴趣与贡献。

本仓库为 [Anduin2017/HowToCook](https://github.com/Anduin2017/HowToCook) 的 fork；欢迎在本 fork 提交 PR，也可直接向上游贡献。

## 简介

HowToCook 是由社区驱动和维护的开源菜谱项目，欢迎贡献新菜谱、改进现有内容或修复问题。

## 快速开始

### 贡献方式

| 方式           | 说明                                 |
| -------------- | ------------------------------------ |
| **修复问题**   | 直接修改并提交 Pull Request          |
| **添加新菜谱** | 复制模板填写后提交（见下方模板链接） |
| **改进文档**   | 完善现有菜谱的描述、步骤或格式       |

### 菜谱模板

写新菜谱时请基于模板修改：[示例菜](./dishes/template/示例菜/示例菜.md)

## 本地开发

### 环境要求

- **Node.js** >= 18.0.0
- **npm**

### 安装与验证

```bash
npm install
npm run build   # 生成 README.md、mkdocs.yml、starsystem/*.md
npm run lint    # 运行全部检查（textlint + markdownlint + manuallint）
```

### 单独执行检查

| 命令                   | 说明                     |
| ---------------------- | ------------------------ |
| `npm run textlint`     | 文本格式（中英文空格等） |
| `npm run markdownlint` | Markdown 格式            |
| `npm run manuallint`   | 手动规则检查             |

## 菜谱编写规范

### 文件结构

```
dishes/
├── vegetable_dish/    # 素菜
├── meat_dish/         # 荤菜
├── aquatic/           # 水产
├── breakfast/         # 早餐
├── staple/            # 主食
├── semi-finished/     # 半成品加工
├── soup/              # 汤与粥
├── drink/             # 饮料
├── condiment/         # 酱料
└── dessert/           # 甜品
```

### 命名规范

- 文件名使用中文，格式为 `菜名.md`
- 如有图片，创建同名文件夹存放：`菜名/菜名.md`
- 标题格式：`# 菜名的做法`

### 必要章节

每个菜谱必须包含以下章节：

1. **简介**：菜品特点、营养价值、预计时长
2. **预估烹饪难度**：1-5 星（使用 ★ 符号）
3. **必备原料和工具**：列出所有必需材料
4. **计算**：给出精确的用量
5. **操作**：详细的烹饪步骤
6. **附加内容**：注意事项、参考资料

### 难度评级标准

| 星级  | 描述               | 预计时间    |
| ----- | ------------------ | ----------- |
| ★     | 简单混合即可       | 约 5 分钟   |
| ★★    | 步骤简单，无需经验 | 约 10 分钟  |
| ★★★   | 需要一定烹饪经验   | 约 15 分钟  |
| ★★★★  | 步骤复杂，需要技巧 | 40 分钟以内 |
| ★★★★★ | 极其复杂，容易失误 | 40 分钟以上 |

### 格式要求

- 中文与英文/数字之间保留一个空格
- 不使用模糊描述（如"适量"、"少量"）
- 使用毫升（ml）作为液体单位
- 需要等待的步骤必须给出时间或判断标准
- 重点内容使用 `*斜体*` 或 `**粗体**` 标注

## Pull Request 规范

### 提交前检查

- [ ] 删除模板中的所有注释
- [ ] 运行 `npm run lint` 确保通过
- [ ] 运行 `npm run build` 更新索引
- [ ] 检查文件放在正确的分类目录

### Commit 信息格式

```
<类型>: <描述>

# 示例
添加: 可乐鸡翅菜谱
修复: 红烧肉步骤描述错误
改进: 优化宫保鸡丁的用料说明
```

类型说明：
- `添加`：新增菜谱
- `修复`：修复错误
- `改进`：改进现有内容
- `文档`：文档相关修改
